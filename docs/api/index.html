DSH REST API (1.0.0)

Download OpenAPI specification:

URL: https://github.com/MRISS-Projects/dsh

Document Smart Highlights (DSH) REST API for document analysis and smart highlighting.

This API allows clients to upload documents, trigger analysis, and retrieve the resulting highlights and metadata.

For detailed feature specifications see:

  • /specs/features/document-analysis.md
  • /specs/features/smart-highlighting.md
  • /specs/features/rest-api-endpoints.md

documents

Document upload, analysis, and management

Upload and analyse a document

Accepts a document file and optional analysis configuration. Returns a document ID and initial processing status. Analysis is performed asynchronously by the indexer worker.

Authorizations:
bearerAuth
Request Body schema: multipart/form-data
required
file
required
string <binary>

Document file (PDF, DOC, DOCX). Maximum size 50 MB.

object (AnalysisOptions)

Configuration options for document analysis

Responses

Local development server

http://localhost:8080/api/v1/documents

Response samples

  • 202
  • 400
  • 413
  • 500
Content type
application/json
{
  • "documentId": "doc-abc123",
  • "status": "processing",
  • "highlights": [ ]
}

highlights

Highlight retrieval and management

Get highlights for a document

Returns all highlights generated for the specified document.

Authorizations:
bearerAuth
path Parameters
documentId
required
string
Example: doc-abc123

Unique identifier of the document

query Parameters
type
string

Filter highlights by type

minConfidence
number [ 0 .. 1 ]
Default: 0.7

Minimum confidence threshold for returned highlights

page
integer >= 0
Default: 0
size
integer [ 1 .. 100 ]
Default: 20

Responses

Local development server

http://localhost:8080/api/v1/documents/{documentId}/highlights
